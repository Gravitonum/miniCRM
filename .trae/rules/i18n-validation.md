# Правило валидации i18n

## Обязательные проверки
- **Никогда не используй ключи без переводов**: Перед добавлением текста в JSX/TSX замени на t('key'), где key описательный (например, 'joinOrg.title' → 'organization.join.title').
- **Проверь все языки**: После изменений:
  1. Найди файлы переводов (src/i18n/, locales/, messages/, public/locales/).
  2. Убедись, что ключ существует во ВСЕХ поддерживаемых языках (ru, en, es и т.д.).
  3. Переводы должны быть полными, естественными, без плейсхолдеров вроде "{{var}}" — используй корректный синтаксис библиотеки (react-i18next/i18next).
- **Если ключ отсутствует**: 
  - Добавь переводы во все JSON/YAML файлы.
  - Тестируй: Переключи язык (i18n.changeLanguage('ru')) и проверь UI на пропавшие ключи.

## Тестирование
- Сгенерируй скриншоты UI для каждого языка.
- Запусти `npm run i18next-scanner` или аналог (если есть) для линтинга.
- Нет пропусков — только тогда применяй diff.

## Пример нарушения
❌ ` <h1>{t('joinOrg.title')}</h1> ` — если 'joinOrg.title' нет в ru.json.
